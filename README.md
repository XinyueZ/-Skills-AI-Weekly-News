## 任务
1. 读取RSS信息来源, 这些来源的URL都定义在“SOURCE_”开头的环境值中，比如SOURCE_0，SOURCE_1，......
2. 要找出**每个信息来源**中离开今天时间最近的5个<entry>但是必须是**7天内**的，多余的不用考虑，使用<entry>中的<published>和<updated>时间戳来依据判断。
3. 随后找出**每个信息来源**中最近的<entry>中的<media:group>, 我们的目标是它的<media:description>中的信息。
4. 结合所有提取的信息（**每个信息来源**中最新的）合成一段适合语音播放的文稿脚本，脚本基于<media:description>中的信息并结合Internet搜索相关议题来**合成丰富**，充实输出的内容，最后提供“简体中文”, “英文”，“德文”，三份不同语言的脚本。
5. 利用合成的文稿脚本生成语音，每个语言一份。
6. 将语音下载为mp3文件，各个语言一份，mp3文件命名规则是根据语言名称然后是扩展名，比如chinese.mp3, german.mp3 .....
7. 保存不同语言的文稿脚本。
8. **严格参考**“输出的index.html的模版“，生成唯一一个index.html页面，页面上可以让我们随意选择三种语言来播放语音，不用太花哨的界面，力求简单好用，底部要有我们预期的链接。
9. Git-commit 所有的mp3文件和index.html文件, 注意，其他文件类型和目录一概忽略（极其重要！）。如果repo中已经存在wrangler.jsonc文件，请务必保留且更新此文件（极其重要！）。
10. Git-push 到 git@github.com:XinyueZ/AI-Weekly-News.git
11. 反思和自我检查:
    - 文稿脚本是否按照规定的数据源为底座？
    - 文稿脚本合成结果是否存在欠缺和遗漏？
    - 所有的输出是否按照规定完成？
    - 是否充分使用了所有的信息来源中的最近数据？
    - html语法通过？
    - 生成的文件名是否符合规范要求？
    - 确认是否只git-commit和push我们规定的文件类型？
    - 检查是否**严格参考**“输出的index.html的模版“制作了index.html页面？
    - 检查index.html底部时候包含了LinkedIn的链接，Github链接和“拷贝入剪切板”的链接，他们都该是图标表示。
    - 反思结果都是正面的话，就结束工作，否则的话，就必须revision。
